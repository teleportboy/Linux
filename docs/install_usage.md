# Установка Debian 12 + вся среда: инструкция

Спека со всеми решениями: `spec_install.md`. Чеклист: `../plan_linux.md`.

## Шаг 1. Debian netinstall (~10 минут руками)

1. Образ: **debian-12.x-amd64-netinst.iso** → на флешку (`dd` / Ventoy / balenaEtcher).
2. Грузишься с флешки → **Install** (можно текстовый, можно Graphical — без разницы).
3. По ходу установщика:
   - язык/локаль — English / United States (русская локаль включится скриптом);
   - **root-пароль — оставить ПУСТЫМ**: тогда инсталлер сам поставит sudo и
     добавит юзера в группу. С заданным root-паролем sudo не ставится вообще —
     скрипт это поймает и подскажет команду, но это лишний перелогин;
   - юзер: **teleport** (имя свободно, скрипт не привязан, но пути в доках под teleport);
   - разметка — как удобно (guided на весь диск ок; своп установщик сделает сам);
   - **Software selection — снять ВСЕ галки десктопов**, оставить только
     `standard system utilities` (+ `SSH server` по желанию — скрипт всё равно поставит).
4. Reboot, вытащить флешку, залогиниться в консоль.

## Шаг 2. Одна кнопка

Если ставил без галки SSH — сеть по кабелю уже работает (DHCP). Дальше:

```sh
sudo apt install -y git
git clone https://github.com/teleportboy/Linux ~/Linux
bash ~/Linux/Debian/install.sh
```

**ВАЖНО: именно `bash`, не `sh`** (см. P5 в спеке). Скрипт спросит пароль sudo
один раз в начале — дальше сам (и поставит NOPASSWD навсегда).

Идёт ~15–40 минут в зависимости от канала. Лог пишется в `~/install-linux.log`.
Упавшие необязательные шаги (сторонние сайты) печатаются в конце списком
«Skipped steps» — их можно догнать повторным запуском: скрипт идемпотентен,
уже сделанное он пропускает/переписывает тем же результатом.

## Шаг 3. После скрипта

```sh
sudo reboot        # группы, microcode, NetworkManager
# login ->
startx
```

- Раскладка us/ru — **Win+Space** (Scroll Lock LED горит на русской).
- Wi-Fi: `nmtui` (или nm-applet в трее полибара).
- Терминал: `$mod+Enter`, закрыть окно: `$mod+q`, меню: `$mod+d`.

## Шаг 4. Ручной перенос со старой машины

| Что | Как |
|-----|-----|
| SSH-ключи и `~/.ssh/config` | скопировать, затем **`chmod 700 ~/.ssh && chmod 600 ~/.ssh/*`** |
| `~/WORKSHOP`, `/opt/kobus9`, `/opt/serverKobus` | rsync/внешний диск |
| `/opt/aarch64-buildroot-linux-gnu_sdk-buildroot` (730 МБ) | оттуда же |
| NXP-тулзы (LinkServer, MCU-Link, lpcscrypt, pemicro) | MCUXpressoInstaller с nxp.com, нужен логин NXP |
| `~/bin/GitExtensions` | скопировать каталог (mono уже стоит) |
| Cursor, RustRover | AppImage/tarball с сайтов → `~/apps` |

## Что изменилось против старой ручной установки

- Раскладка переключается **Win+Space**, правый Ctrl больше ничего не делает
  (решение 2026-07-23); из-за этого в i3 отключён дефолтный бинд
  `$mod+space` (focus mode_toggle).
- Конфиги приезжают готовыми файлами из `Debian/home/` — править их теперь
  правильно так: поменял на машине → скопировал в репо → закоммитил.
- oh-my-zsh ставится без вопросов, `startx` руками после reboot (в скрипте его нет).
- NFS-сервер по умолчанию НЕ ставится (`INSTALL_NFS_SERVER=0` в шапке скрипта).
- Ненужные секции выключаются флагами в шапке `install.sh` до запуска.
- Повторный запуск безопасен и на уже настроенной машине, но перезапишет
  дотфайлы версиями из репо — репо источник истины (правки конфигов носи
  в обратную сторону: машина → репо → коммит).
